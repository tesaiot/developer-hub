# Multi-Screen Smart Watch

## เกี่ยวกับตัวอย่างนี้
นาฬิกาอัจฉริยะแบบหน้าจอกลม มี 4 หน้าปัดที่เลื่อนด้วยการปัดซ้าย-ขวา (Swipe Navigation) ประกอบด้วย Clock (นาฬิกาดิจิทัล), Sensors (ข้อมูล IMU/อุณหภูมิ/ความชื้น), Steps (นับก้าวด้วย arc gauge), Weather (สภาพอากาศจากเซ็นเซอร์) ข้อมูลเซ็นเซอร์ทั้งหมดอ่านโดยตรงจาก app_sensor

## สิ่งที่จะได้เรียนรู้
- การสร้าง tileview แบบปัดในแนวนอนภายใน container ทรงกลม
- การจัดโครงสร้างโปรเจกต์แบบ multi-file (แยกไฟล์ต่อหน้าจอ)
- การอ่านเซ็นเซอร์ 4 ตัวผ่าน `app_sensor` reader API
- การจำลองการนับก้าวจาก accelerometer magnitude
- การคำนวณระดับความสูงจากความดันอากาศ (ISA formula)
- การใช้ `lv_arc_create()` แสดง gauge แบบวงกลม

## โครงสร้างไฟล์
| ไฟล์ | หน้าที่ |
| --- | --- |
| `main_example.c` | จุดเริ่มต้น, init เซ็นเซอร์, timer callbacks |
| `watch_screens.h` | Forward declarations สำหรับแต่ละหน้าจอ |
| `watch_clock.c` | หน้าจอนาฬิกาดิจิทัล |
| `watch_sensors.c` | หน้าจอข้อมูลเซ็นเซอร์ IMU/Temp/Hum |
| `watch_steps.c` | หน้าจอนับก้าวด้วย arc gauge |
| `watch_weather.c` | หน้าจอสภาพอากาศ |

## วิธีติดตั้ง
1. คัดลอกไฟล์ทั้งหมดในโฟลเดอร์นี้ไปที่ `proj_cm55/apps/`
2. ตรวจสอบว่า `app_sensor/` อยู่ใน include path
3. Build และ flash ตามปกติ

## สิ่งที่จะเห็นบนหน้าจอ
หน้าจอกลมขนาด 280x280 พิกเซลมีขอบสีฟ้า ด้านล่างมี dot indicator แสดงตำแหน่งหน้าปัจจุบัน ปัดซ้ายเพื่อเปลี่ยนไปหน้าถัดไป ข้อมูลเซ็นเซอร์อัพเดททุก 200ms นาฬิกาอัพเดททุก 1 วินาที

## ลองปรับแต่ง
- เพิ่มหน้าปัดที่ 5 แสดงข้อมูลเข็มทิศจาก BMM350
- เปลี่ยนการนับก้าวให้ใช้ algorithm ที่แม่นยำกว่า
- เพิ่ม animation เมื่อเปลี่ยนหน้าปัด
- ปรับ arc gauge ให้แสดงสีตาม progress (เขียว/เหลือง/แดง)
